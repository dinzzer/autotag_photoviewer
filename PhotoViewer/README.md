# PhotoViewer — Google Photos 風ローカル写真ビューワー (Android)

仕様書 `Google_Photos_Style_Viewer_Specification.md` に基づく、ローカル写真の閲覧体験に特化した
Android アプリです。**Kotlin + Jetpack Compose** で実装し、指に吸い付くアニメーション・共有要素遷移・
ピンチによる表示密度切り替えなど「手触り」を重視しています。

このマイルストーンの範囲は **コア閲覧体験** です（クリックで決定したスコープ）。

## 動作要件

- Android Studio Ladybug (2024.2) 以降
- JDK 17（Android Studio 同梱の JBR で可）
- Android Gradle Plugin 8.7 / Gradle 8.9 / Kotlin 2.0.21
- 実機または API 24 以上のエミュレータ（写真は端末の MediaStore から読み込み）

## ビルド方法

1. Android Studio で本フォルダ（`PhotoViewer`）を **Open** する。
2. 初回同期で依存関係が自動取得される。`local.properties` は Studio が自動生成する。
3. 実機/エミュレータを選んで **Run**。
4. 起動時に写真へのアクセス許可を求められるので許可する
   （Android 13+ は「写真と動画」権限、12 以下は「ストレージ」権限）。

> コマンドラインでビルドしたい場合は、一度 Gradle を入れて
> `gradle wrapper --gradle-version 8.9` を実行すると `gradlew` 一式が生成されます。
> （リポジトリにはバイナリの `gradle-wrapper.jar` を含めていません）

## 実装済みの機能

### タイムライン画面 (`ui/timeline`)
- **日付グルーピング**: EXIF 撮影日時（`DATE_TAKEN`、無ければ `DATE_ADDED`）で日別グリッド。
  各グループ先頭に「6月14日(日)」形式の日付ヘッダー。
- **表示スケール切替（ピンチ）**: 快適(3列) / コンパクト(6列) / 月別(15列) の3層。
  ピンチ中はグリッド全体がリアルタイムに拡縮し、閾値超えで次の層へスナップ、
  指を離すとスプリング（ゴムが縮む弾性）で 1.0 倍に戻る。月別層は月単位グルーピングに変化。
- **ヘッダーの自動フェード**: 下スクロールで上部ヘッダーが隠れ、上スクロールで再表示。
- **快速シーク スクロールバー**: 右端のサムをドラッグすると年月バブルがポップ表示され、
  一覧位置へ即ジャンプ。
- **ドラッグ複数選択・一括整理（仕様4.6）**: 写真を長押しで選択モードに入り（ハプティクス）、
  そのまま指でなぞると軌跡上の写真が次々と選択（チェックマークがバウンスでポップ、選択写真は
  わずかに縮小）。画面の上端／下端にドラッグを保持すると、端への近さに応じて加速しながら
  オートスクロール。上部の選択バーに選択枚数・解除（×）・一括削除。削除はAndroid標準の
  削除確認ダイアログ（`MediaStore.createDeleteRequest`, API 30+）経由。システムバックで選択解除。

### ライトボックス画面 (`ui/lightbox`)
- **共有要素（ヒーロー）遷移**: サムネイルをタップすると、その写真が全画面へ滑らかに拡大。
  `SharedTransitionLayout` + `AnimatedContent` による Material 標準の遷移。
- **スワイプ移動**: `HorizontalPager` で左右に写真送り。写真間に余白（`pageSpacing`）あり。
- **ダブルタップ / ピンチズーム**: ダブルタップで「タップ位置」を中心に 2.5 倍へズーム、
  もう一度で全体表示。ピンチで 1〜5 倍の自由ズーム＋パン（端でクランプ）。
- **スワイプダウンで閉じる**: 等倍時に下方向ドラッグで写真が縮小し背景の黒が透過、
  画面高の 20% を超えて離すと共有要素遷移でサムネイル位置へ吸い込まれて閉じる。
  閾値未満ならスプリングで全画面に戻る。
- **オーバーレイ UI**: 画面タップで戻る/撮影日時/削除アイコンがフェードイン・アウト。
- **スワイプ仕様**: 写真を**上方向**にスワイプするとタグパネルが出現（詳細表示の機能）。
  詳細を閉じて一覧へ戻るのは**下方向**スワイプのみ。左右スワイプは写真送り。
- **タグパネルと編集モード**: タグはオンデバイス自動タグ付け（仕様 6）のキャッシュ結果。
  「編集」で編集モードに入り、各タグに×が出てタップで削除、テキスト入力での追加、
  既知タグの候補チップから選んで追加ができる。手動編集した写真は自動タグ付けで上書きされない。

### ナビゲーション（下部タブ）
- **フォト**: 日付順の全画像タイムライン。
- **検索（アルバム）**: Google Photos のアルバム風。タグ未選択時は、各タグの代表画像（最も
  確信度の高い写真）＋タグ名＋枚数のカード一覧を表示。カードをタップするとそのタグの写真一覧へ。
  写真一覧の上部にはフィルタバー（テキスト入力＝既知タグのインクリメンタル絞り込み＋任意語の確定追加、
  候補チップ、**AND/OR 切替**）があり、さらに絞り込める。選択タグが増えても見やすいよう、選択タグ・
  候補チップ列は**結果のスクロールに合わせて自動フェード**（上スクロールで再表示）。システムバック／
  「アルバムへ」でアルバム一覧に戻る。アクティブなタグエンジン名（例: WD Tagger v3 / スタブ）も表示。

### スクロール位置の保持
詳細表示（ライトボックス）を下スワイプで閉じると、一覧の先頭ではなく**写真をタップした元の
スクロール位置**へ戻る（グリッド状態をタブ上位に巻き上げて保持。ヒーロー遷移の戻り先サムネイルも一致）。

### タグの永続化
Room はバージョン更新時も**破壊的移行を行わず**（`MIGRATION_1_2`）、再ビルド／アプリ更新でも既存タグは
保持される。バックグラウンドのタグ付けは `tagged_photos` に未登録の写真（＝未分類）だけを対象に再開する。
（アプリを**アンインストール**するとアプリデータごと消えるため、その場合のみ作り直しになります）

### パフォーマンス（仕様 5）
- アプリ独自の Coil `ImageLoader`（`PhotoApp`）でメモリ（RAMの25%）／ディスク（256MB）キャッシュを確保し、
  クロスフェード 150ms。高速スクロールでもキャッシュ済みサムネを即時表示し、白いプレースホルダ時間を短縮。
- グリッドのサムネはセルサイズに応じた解像度でデコード（快適512 / コンパクト256 / 月別128px）。
  フル解像度を毎回デコードしないため、スクロールとピンチが軽快。

### 戻るボタン（システムバック）の階層
直接アプリを終了せず、メイン画面へ戻る動作を経由する。詳細表示中はタグパネルを閉じる → 詳細を閉じて一覧へ。
検索タブでは絞り込み解除 → アルバム一覧 → フォトタブへ。フォトタブ最上位で初めてアプリ終了。
選択モード中のバックは選択解除。

### 触覚フィードバック
- 以下のタイミングで微細な振動を発火: 長押しで選択モードに入った瞬間／ピンチでスケール層が
  切り替わった瞬間／ダブルタップズーム／スワイプダウンで閉じる確定。

## アーキテクチャ

```
data/
  Photo.kt                 写真モデル
  MediaStoreRepository.kt  MediaStore から画像を読み込み（撮影日降順）
  tagging/AutoTagger.kt    自動タグ付けのインターフェース + スタブ実装
ui/
  PhotoViewerApp.kt        権限ゲート + SharedTransitionLayout で2画面を結線
  theme/Theme.kt           ダーク基調の Material3 テーマ
  timeline/                タイムライン画面・グルーピング・ViewModel
  lightbox/                ライトボックス画面・ズーム可能画像
MainActivity.kt            エントリポイント（edge-to-edge）
```

画像読み込みは **Coil**、権限は **Accompanist Permissions** を利用。

## 自動タグ付け（仕様 6）

danbooru / wd-tagger 系の ONNX モデルを **ONNX Runtime Mobile** でオンデバイス推論し、結果を
**Room** にキャッシュします。タイムライン上部のタグチップで絞り込み（複数選択は AND 条件）、
ライトボックスのタグパネルはキャッシュ済みタグを表示します。写真読み込み後に未処理の写真を
バックグラウンドで逐次タグ付けし、進捗をヘッダーに表示します。

構成:
- `data/tagging/AutoTagger.kt` … 共通インターフェース＋`StubAutoTagger`（モデル未配置時のフォールバック）。
- `data/tagging/OnnxAutoTagger.kt` … wd-tagger 標準前処理（正方形パディング→448 リサイズ→BGR float、
  NHWC）＋しきい値でタグ抽出。`selected_tags.csv` をラベルに使用。
- `data/tagging/TaggerFactory.kt` … モデル有無を判定し ONNX かスタブを選択。
- `data/tagging/TagRepository.kt` … 逐次タグ付け・キャッシュ・各種 Flow。
- `data/db/` … Room（`photo_tags` / `tagged_photos`）。

### モデルの配置（重要）

モデルは大きいためリポジトリには同梱していません。次のいずれかに 2 ファイルを置くと
`OnnxAutoTagger` が有効化され、無ければ自動的に `StubAutoTagger` にフォールバックします。

1. ビルド時に同梱: `app/src/main/assets/model.onnx` と `app/src/main/assets/selected_tags.csv`
2. 実機に後から push（再ビルド不要）:
   `adb push model.onnx /data/local/tmp/ && adb shell run-as com.dinz.photoviewer cp /data/local/tmp/model.onnx files/`
   （`selected_tags.csv` も同様。`files/` は `Context.filesDir`）

モデル例: SmilingWolf 系 wd-v1-4 / wd-v3 の ONNX（入力 448、出力は確率）。入力サイズ・RGB/BGR・
しきい値はモデルに合わせて `OnnxAutoTagger` のコンストラクタ引数で調整してください。

## 既知の制限 / 今後の拡張余地

- ズーム中に写真端へ達したら次の写真へ、という連続コンテキスト移行（仕様 4.4）は、
  現状「ズーム中はページ送りを無効化」する簡易版。
- 複数選択（仕様 4.6）は、長押しで選択モードへ入った後いったん指を離し、改めてドラッグして
  なぞる方式（長押し→指を離さず連続ドラッグの完全な一筆操作は未対応）。一括削除は API 30+ の
  システム削除ダイアログ経由（API 29 以下は contentResolver の best-effort 削除）。
- 超高速スクロール時の EXIF 埋め込みサムネイル即時表示（仕様 5）は Coil のキャッシュ＋
  クロスフェードで近似。専用の二段サムネイル差し替えは未実装。
- Android 14 の「一部の写真のみ許可」への最適化は簡易対応。
- 削除アイコンは UI のみ（`MediaStore` 削除 Intent への結線は未実装）。
